How the Data Is Reduced

This page summarizes the reduction workflow used to produce cleaned Chandra cluster data products for the archive.

Reduction workflow

  1. Start from publicly available Level 1 event files from the Chandra archive for each ObsID.
  2. Reprocess observations with standard CIAO/CALDB calibration updates to generate clean Level 2 event files.
  3. Apply flare filtering and good-time interval screening to remove high-background periods.
  4. Detect and mask point sources before generating diffuse cluster products.
  5. Create exposure-corrected imaging products and export cleaned FITS files with metadata for archive use.
  6. Run quality checks on astrometry, exposure maps, and file integrity before publication.
